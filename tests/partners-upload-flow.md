# Тест: завантаження логотипів партнерів

## Мета
Переконатися, що адмін може завантажити лого партнера до Firebase Storage, записати doc у `partners` та негайно побачити відображення у секції.

## Передумови
- У `firebaseConfig` заповнені ключі, `ADMIN_UIDS` містить ваш UID.
- Увімкнено Firestore та Storage у проекті Firebase.

## Кроки
1. Відкрити `public/index.html` через локальний сервер (наприклад, `python -m http.server 8080 --directory public`).
2. Натиснути "⚙️ Admin" → увійти через Google (UID має бути у `ADMIN_UIDS`).
3. У формі "Partners" вказати назву і, за бажанням, URL.
4. У дропзоні "Лого" обрати/перетягнути зображення (image/*).
5. Натиснути "Додати".
6. Переконатися у Network/Console, що:
   - Виконався upload у Storage (шлях `partners/<timestamp>_<filename>`).
   - Firestore отримав новий документ у колекції `partners` з полем `logo_gs_url`.
7. Перевірити секцію "Partners & Friends" на сторінці — нове лого відображається одразу (через `downloadURL`).
8. Перезавантажити сторінку й переконатися, що лого підтягується через gs→https конвертацію без 404/CORS.

## Очікуваний результат
- Кнопка після сабміту повертає попередній текст (немає зависань).
- У секції партнери видно новий елемент з лого.
- У вкладці Network немає повторюваних 404/CORS для цього лого після перезавантаження.
