# Тести: data-driven рендер

## Мета
Переконатися, що сторінка коректно відображає дані з Firestore або мок-даних, не падає без реальної конфігурації Firebase, та працюють фільтри релізів.

## Передумови
- `public/index.html` відкрито в браузері.
- Якщо реальна Firebase-конфігурація не задана, сторінка використовує вбудовані мок-дані (консоль попереджає про відсутність config).

## Кроки
1) Hero/About
   - Очікування: заголовок і біо заповнені (мок або Firestore `artist_info`).
2) Discography
   - Є хоча б одна картка. Клік по картці відкриває модалку зі Spotify iframe (мок) або embed з Firestore.
   - Перемикання фільтрів Albums/Singles/Remixes: грид оновлюється (якщо типи присутні), без помилок у консоль.
3) Videos
   - Є картка відео; клік відкриває модалку з YouTube iframe.
4) Photos
   - Є картки фото; клік відкриває модалку з зображенням.
5) Events
   - Список містить хоча б один запис; відсутність даних показує текст "Наразі немає подій".
6) Label
   - Блок з описом/лінками відображається; за відсутності даних показує текст-заглушку.
7) Partners
   - Є картки партнерів або текст про відсутність даних; без помилок у консоль.
8) Contact
   - Є email-лінк і соц-іконки (з моків або Firestore socials). Лінки клікабельні.

## Результат
- Pass, якщо всі очікування виконані. Якщо використовується реальна Firebase, перевірити, що дані відповідають Firestore. Якщо мок — головне, щоб UI не падав і контент показується/модалки працюють.
