# Тест: відсутній файл у Storage (уникнення 404)

## Мета
Переконатися, що за відсутності файлу у Firebase Storage UI не генерує нескінченні 404/CORS запити, а просто не показує зображення.

## Кроки
1) У Firestore залишити запис із `hero_image` або `image_gs_url`, що вказує на неіснуючий файл у bucket.
2) Відкрити сторінку через локальний сервер.
3) Очікування:
   - У Network немає повторюваних 404/CORS для цього шляху (URL не запитується, оскільки повертається `null`).
   - Відповідна картка/секція просто без зображення, без зависань UI.

## Результат
- Pass, якщо 404/CORS не спамується і UI працює без зависання.
