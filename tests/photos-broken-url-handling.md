# Тест: пропуск проблемних фото

## Мета
Переконатися, що фото з відомими проблемними URL не рендеряться повторно та не спричиняють нескінченних 404/CORS.

## Кроки
1. Запустити локально `public/index.html` (наприклад, `python -m http.server 8080` з кореня та відкрити `http://localhost:8080/public/`).
2. У Firestore або мок-даних вказати у колекції `photos` запис із невалідним `url`.
3. Дочекатися, поки зображення зламається (перевірити у DevTools, що URL додано до `brokenUrls`).
4. Перезавантажити сторінку.
5. Перевірити, що картка фото з цим URL не містить `<img>` і не викликає нових запитів до проблемного ресурсу (в Network табі немає повторних 404/CORS на цей URL).

## Очікуваний результат
- Після першої фіксації помилки зображення більше не вставляється в DOM для цього URL.
- Вкладка Network не показує повторних запитів до того ж проблемного URL після перезавантаження/переходу по секції.
