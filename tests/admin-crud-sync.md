# Тести: синхронізація CRUD без моків

## Мета
Переконатися, що при ввімкненому Firebase дані одразу видно на фронтенді після додавання/видалення, а моки не підміняють реальний вміст.

## Передумови
- Указано дійсний firebaseConfig.
- UID користувача в ADMIN_UIDS.
- Firestore містить тестову колекцію (може бути порожня).

## Кроки
1) Вхід
   - Відкрити адмін-панель, авторизуватись (Google). Статус: онлайн.
2) Додавання релізу
   - Заповнити Title, Type, Spotify. Натиснути «Додати».
   - Очікування: картка релізу з'являється одразу (без оновлення сторінки), тост «Реліз додано».
3) Видалення релізу
   - Натиснути «Видалити» на щойно доданій картці.
   - Очікування: картка одразу зникає, тост «Видалено».
4) Перевірка моків
   - Якщо колекція спорожніла, у грідах показується порожній стан (без демо-даних). Мок-контент не підтягується.
5) Аналогічно перевірити Videos/Photos/Partners (додати → одразу видно; видалити → одразу зникає).

## Результат
- Pass, якщо всі CRUD дії відображаються миттєво, без підміни моками, і порожні колекції показують пустий стан.
