# Тести: адмін режим, авторизація, CRUD, upload

## Мета
Переконатися, що адмін-режим працює через Google sign-in, права гейтяться по `ADMIN_UIDS`, CRUD-операції для всіх колекцій виконуються, drag-and-drop upload у Storage працює, а UI оновлюється в реальному часі.

## Передумови
- У `public/index.html` заповнено `firebaseConfig` реальними ключами.
- У `ADMIN_UIDS` додано UID акаунта тестового адміна.
- Firestore Rules дозволяють запис лише для цих UID (як у спеці):
  ```
  allow write: if request.auth != null && request.auth.uid in ['YOUR_ADMIN_UID'];
  ```

## Кроки
1) Авторизація
   - Натиснути "⚙️ Admin". Авторизуватися через Google.
   - Очікування: повідомлення в шапці адмін-панелі показує email та статус адміна; панель відображається.
2) Artist info
   - Заповнити tagline/bio/email/socials, за потреби додати hero image через drag&drop.
   - Submit. Очікування: без помилок; у Hero/About з'являються нові тексти; контактний email/соцмережі оновлюються.
3) Releases
   - Додати реліз (title/date/type/spotify URL) і завантажити обкладинку через dropzone.
   - Очікування: картка з'являється у Discography без перезавантаження; клік відкриває модалку з Spotify iframe.
4) Videos
   - Додати відео (title + YouTube ID) і прев'ю (опц.) через dropzone.
   - Очікування: картка у Videos, модалка з YouTube iframe.
5) Photos
   - Додати фото (alt/category) і файл через dropzone.
   - Очікування: картка у Photos, модалка показує зображення.
6) Events
   - Додати подію (title/date/location/ticketUrl, past optional).
   - Очікування: рядок у Events (сортування за датою збережеться), квитковий лінк клікабельний.
7) Label
   - Оновити назву/опис/лінки, додати лого через dropzone.
   - Очікування: секція Label оновлена, лого відображається.
8) Partners
   - Додати партнера (name/url) з лого через dropzone.
   - Очікування: картка партнера з лого, hover-глоу.
9) Видалення
   - Натиснути "Видалити" на релізі/відео/фото/події/партнері.
   - Очікування: елемент зникає в реальному часі, без помилок у консоль.
10) Вихід
    - Натиснути "Вийти" в адмін-панелі.
    - Очікування: панель ховається, статус змінюється на "Увійдіть".

## Результат
- Pass, якщо всі дії проходять без помилок/консольних виключень, дані в Firestore/Storage з'являються, UI оновлюється в реальному часі.

All Fail - спрацювала тільки авторизація та вихід
