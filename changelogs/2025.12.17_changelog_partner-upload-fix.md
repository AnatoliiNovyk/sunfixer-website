# Зміни 2025-12-17

- Виправлено хендлер додавання партнера: завантаження логотипу тепер зберігається у Storage з коректним `logo_gs_url`, документ пишеться у колекцію `partners`, а локальний стан оновлюється `downloadURL` для миттєвого відображення.
- Було: код падав через неіснуючу змінну `logo_gs_url` та записувався у колекцію `photos`, через що аплоад зривався і UI не оновлювався.
- Стало: аплоад проходить, партнер додається у `partners`, логотип одразу рендериться; при повторному завантаженні сторінки лого підтягується через gs→https конвертацію без зайвих 404/CORS.
- Додано тестовий сценарій `tests/partners-upload-flow.md` для перевірки аплоаду логотипів партнерів і миттєвого відображення.
- Причина: основна проблема — неможливість завантажити лого через адмінку — була у некоректній логіці збереження; тепер це усунено.
