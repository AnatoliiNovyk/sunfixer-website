# Changelog 2025.12.16 — Firebase каркас і data-driven рендер

## Було
- Каркас сторінки з демо-модалками без підключення до даних чи Firebase.

## Стало
- Підключено CDN Firebase compat (app/auth/firestore/storage) з placeholder-конфігом.
- Додано константу `ADMIN_UIDS` (UI-гейт для майбутнього адмін-режиму).
- Реалізовано стан та рендер усіх секцій з даних: Hero/About, Discography (фільтри), Videos, Photos, Events, Label, Partners, Contact.
- Реалізовано fallback на мок-дані, якщо Firebase config не заповнено, щоб сторінка працювала без секретів.
- Модалки тепер будуються з реєстру контенту (release/video/photo) на основі даних.
- Додано тестовий файл [tests/data-render.md](../tests/data-render.md) для перевірки рендеру/модалок з даними.

## Причина
- Виконання Кроку 4 плану: інтегрувати Firebase compat, підготувати data-driven рендер і безпечний fallback без реальних ключів.

## Примітки/тести
- Для реальної Firebase потрібно заповнити `firebaseConfig` у `public/index.html`. Без цього автоматично підтягуються мок-дані (консоль попереджає).
- Ручні тести описані у `tests/data-render.md`. Автоматизація не додавалася.
