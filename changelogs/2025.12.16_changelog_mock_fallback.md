# Changelog — мок-дані при порожньому Firestore

## Що зроблено
- Додано прапорець `ENABLE_MOCK_IF_EMPTY` у `public/index.html`, який автоматично вмикає демо-дані, якщо колекції Firestore порожні.
- Додано відстеження `state.usedMock` і одноразове підключення моків для секцій (artist_info, releases, videos, photos, events, label, partners) із консольними попередженнями.

## Як було
- Якщо Firebase-конфіг заповнено, але Firestore порожній, UI виглядав порожнім без демо-контенту, що заважало проходити чеклісти.

## Як стало
- При порожніх колекціях автоматично показуються демо-дані; коли в Firestore з’являться записи, вони замінять моки без перезавантаження.

## Причина змін
- Дати змогу проходити чеклісти та демо-огляди навіть за відсутності реальних записів у Firestore.
