# Changelog 2025.12.17_1248 — уникнення 404/CORS для відсутніх файлів

## Зміни
- `getDownloadUrlIfGs` тепер повертає `null`, якщо файл у Storage недоступний/відсутній, замість генерації fallback URL. Це прибирає повторні 404/CORS запити, коли об'єкта реально немає.
- Додано тест-кейс для перевірки поведінки при відсутньому файлі.

## Як було
- При помилці `getDownloadURL` формувався HTTPS-шлях, який все одно віддавав 404/CORS, продовжуючи спамити Network.

## Як стало
- Для недоступних файлів зображення просто не показується, додаткові запити не йдуть.

## Причина
- Уникнути "зависань" через постійні 404/CORS, коли файла в bucket немає або доступ закритий.
