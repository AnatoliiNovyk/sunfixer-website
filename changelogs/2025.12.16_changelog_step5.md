# Changelog 2025.12.16 — Admin режим, CRUD, Upload

## Було
- Data-driven рендер з Firebase/mok, без авторизації та CRUD/аплоадів.

## Стало
- Додано адмін-панель (увімкнення через "⚙️ Admin"): Google sign-in, гейтінг по `ADMIN_UIDS`, статуси/нотифікації.
- Реалізовано CRUD UI для всіх колекцій: artist_info, releases, videos, photos, events, label, partners.
- Додано drag-and-drop upload у Firebase Storage для обкладинок/прев’ю/фото/лого; запис `gs://` шляхів у Firestore.
- Картки/списки підтримують видалення (кнопки "Видалити"/✕ при активному адмін-режимі).
- Адмін-панель вимикається, якщо Firebase config не заповнено; fallback мок-даних збережено.
- Оновлено модалку на роботу з реєстром контенту (release/video/photo) з даних.
- Додано тестовий файл [tests/admin-flow.md](../tests/admin-flow.md) з перевірками авторизації, CRUD і upload.

## Причина
- Виконання Кроку 5 плану: впровадити адмін-режим із правами, CRUD і drag&drop завантаженням у Storage з live refresh.

## Примітки/тести
- Для реальної роботи потрібен заповнений `firebaseConfig` та UID у `ADMIN_UIDS`; Firestore Rules мають дозволяти записи лише цим UID.
- Ручні тести описані у `tests/admin-flow.md`; автоматизація не додавалась.
