# Changelog 2025.12.16_1232 — CRUD синхронізація без моків

## Зміни
- Прибрано автопідміну мок-даних у onSnapshot при порожніх колекціях; порожній стан показується без демо-контенту.
- Оптимістичне оновлення state/render для додавання та видалення релізів, відео, фото, подій і партнерів з тостами успіху/помилки.
- Фолбек моку обмежено до випадку, коли Firebase не готовий; додано попередження при спробі CRUD без конфігу.
- Додано тестовий сценарій перевірки синхронізації CRUD без моків.

## Як було
- Порожні снапшоти або таймаут могли включати мок-дані, через що після delete/add фронт «відкочувався» до демо-контенту.
- CRUD покладався лише на Firestore snapshots; UI не оновлювався одразу й помилки видалення не мали тостів.

## Як стало
- Порожні колекції показують пустий стан, без примусового мок-контенту.
- Додавання/видалення одразу оновлює локальний state та UI, успішні операції підтверджуються тостами.
- Моки більше не накладаються при наявності Firebase; CRUD у мок-режимі блокується попередженням.
- Є окремий тестовий чекліст для перевірки синхронізації CRUD.

## Причина
- Забезпечити видимість реальних CRUD-операцій на фронтенді та уникнути перекриття мок-даними.
